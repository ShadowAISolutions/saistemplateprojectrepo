### Key decisions made
- **Content-addressable server-side lookup beats client-side positional indexing** — `writeCell(row, col)` addresses cells by client-side row index, which can drift out of sync with the sheet (sorts, concurrent edits, row deletions between modal open and Save). `addRow` with merge-on-barcode is content-addressable on the server: it looks up the target row by barcode value in real-time. The pivot in v10.72r was recognizing that the problem wasn't "how do I update a cell?" but "how do I update a row by its semantic identifier?" The barcode IS the semantic identifier for inventory rows
- **Reuse proven paths over reinventing machinery** — v10.70r tried to build a parallel write path (writeCell chain + optimistic update). v10.72r realized the scan-existing path already does exactly what's needed and pivoted to reuse it via barcode-scan synthesis. The user's hint was a gentle reminder that I should have looked for existing working machinery before building new code
- **Cosmetic labels are cheap, behavioral identity is precious** — v10.72r kept "Edit Row — {name}" title and "Save" button text (vs scan-existing's "Existing Item — {name}" + "Add Row") via a one-line `isEditAction = scannedFormat === 'EDIT'` check. Everything else routes through the exact same code. This is the minimal divergence needed to distinguish edit actions from scan actions in the UI while reusing 100% of the behavior
- **Defensive improvements are worth keeping even when the original motivation disappears** — the `.onclick` listener management fix from v10.71r was added to try to fix the edit bug. It didn't fix that bug (the real issue was deeper), but it IS a general-purpose improvement that prevents listener stacking for ANY modal-reopen scenario. Kept it in v10.72r even though the edit-mode pivot made it orthogonal to the fix
- **Dead code cleanup when evidence supports it** — v10.69r removed ~2 lines of dead code (old Last User/Last Updated auto-population lines) that became unreachable after the early-continue intercept. v10.72r removed ~90 lines of dead code (the entire editOptions/isEdit branches) after the pivot made them obsolete. Both followed the Dead Code Detection Methodology — trace all consumers, verify unreachability, remove
