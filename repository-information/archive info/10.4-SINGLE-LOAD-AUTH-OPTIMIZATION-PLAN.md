# 10.4 — Single-Load Auth Optimization Plan

> **Goal:** Reduce the standard (URL exchange) login flow from 2 `doGet()` calls to 1, cutting one full iframe reload per sign-in.

> **Status:** Ready to implement
> **Prerequisite:** Phases 1–7, 9 complete (implement BEFORE Phase 8 CSP, since this modifies inline `<script>` blocks)
> **Affects:** `testauth1.gs` (GAS), `testauth1.html` (HTML)
> **Scope:** Standard path only — the HIPAA (postMessage) path is unchanged

## 1. What This Plan Does

### The problem

Every standard-path login triggers **two separate `doGet()` calls**:

| Load | URL | Purpose |
|------|-----|---------|
| Load 1 | `?exchangeToken=<googleToken>` | Exchange OAuth token → create session → return a **tiny HTML stub** that only posts `gas-session-created` |
| Load 2 | `?session=<sessionToken>` | Validate session → return **full app HTML** with `signAppMessage()` call |

Load 1 already has everything needed to build the full app HTML — it just discards it and returns a stub. The HTML parent receives `gas-session-created`, saves the session, then reloads the iframe with `?session=` to get the actual UI. That reload is unnecessary.

### The fix

Make Load 1 return the full app HTML directly (with `gas-session-created` embedded). The parent sees `appLoaded: true` in the message and skips the iframe reload.

### What does NOT change

- **HIPAA (postMessage) path** — unchanged. Its listener-page architecture inherently requires a second load
- **Session resume** (page refresh) — still uses `?session=` as before
- **Tab reclaim** ("Use Here") — still uses `?session=` as before
- **All security mechanisms** — HMAC signing, nonce verification, origin validation, audit logging

### Standard path (`TOKEN_EXCHANGE_METHOD: 'url'`)

```
User clicks "Sign In" → Google OAuth → handleTokenResponse()
    │
    ▼
┌─ LOAD 1: iframe.src = baseUrl + '?exchangeToken=' + googleToken ─────────┐
│  GAS doGet():                                                             │
│    exchangeTokenForSession(googleToken)                                   │
│      ├─ Rate limit check (CacheService)                                   │
│      ├─ validateGoogleToken() (UrlFetchApp → Google tokeninfo)            │
│      ├─ Domain restriction check                                          │
│      ├─ ACL check (SpreadsheetApp)                                        │
│      ├─ invalidateAllSessions(email)                                      │
│      ├─ Generate sessionToken (48-char) + messageKey (32-char)            │
│      ├─ Store session in CacheService (with HMAC integrity)               │
│      └─ auditLog() × 2                                                    │
│                                                                           │
│  Returns: tiny HTML stub (lines 1352–1357 in testauth1.gs)               │
│    → window.top.postMessage({type: 'gas-session-created', ...}, ORIGIN)   │
└───────────────────────────────────────────────────────────────────────────┘
    │
    ▼  HTML receives gas-session-created (lines 1457–1517 in testauth1.html):
    │    ├─ _importHmacKey(messageKey)
    │    ├─ saveSession(token, email)
    │    ├─ _pendingSessionShow = {email, displayName}
    │    └─ _expectingSession = true
    │
    ▼  IFRAME RELOAD (line 1516)
┌─ LOAD 2: iframe.src = baseUrl + '?session=' + sessionToken ──────────────┐
│  GAS doGet():                                                             │
│    validateSession(sessionToken) (lines 631–695)                          │
│      ├─ CacheService.get (read session)                                   │
│      ├─ HMAC integrity check                                              │
│      ├─ Absolute + rolling timeout checks                                 │
│      └─ Update lastActivity + re-store                                    │
│                                                                           │
│  Builds full app HTML (lines 1423–1604):                                  │
│    ├─ Embeds _sessionToken, user email, version                           │
│    ├─ google.script.run.signAppMessage() → posts gas-auth-ok              │
│    └─ Registers listeners (version-check, user-activity, save-note)       │
└───────────────────────────────────────────────────────────────────────────┘
    │
    ▼  HTML receives gas-auth-ok (lines 1557–1605):
    │    ├─ showApp(email, displayName)
    │    ├─ startCountdownTimers()
    │    ├─ startHeartbeat()
    │    └─ broadcastTabClaim()
    │
    ▼  ✅ APP VISIBLE
```

**Total: 2 `doGet()` + 1 `google.script.run` = 3 GAS executions per login**

### HIPAA path (`TOKEN_EXCHANGE_METHOD: 'postMessage'`)

```
LOAD 0: iframe.src = baseUrl (no params)
  → Returns listener page (lines 1362–1400)
  → Waits for 'exchange-token' postMessage
  → Calls google.script.run.exchangeTokenForSession()
  → Posts gas-session-created back to parent

LOAD 1: iframe.src = baseUrl + '?session=' + sessionToken
  → Same as standard path Load 2 above
```

**Total: 2 `doGet()` + 1 `google.script.run` (exchange) + 1 `google.script.run` (signAppMessage) = 4 GAS executions**

This path stays at 2 loads because the first load is a listener page — it can't transform into the app HTML. `google.script.run` returns data, not HTML, so the iframe content can't be replaced.

## 3. Optimized Architecture

### Standard path — after optimization

```
User clicks "Sign In" → Google OAuth → handleTokenResponse()
    │
    ▼
┌─ SINGLE LOAD: iframe.src = baseUrl + '?exchangeToken=' + googleToken ────┐
│  GAS doGet():                                                             │
│    exchangeTokenForSession(googleToken)                                   │
│      └─ (all existing checks — unchanged)                                 │
│                                                                           │
│  IF success:                                                              │
│    validateSession(sessionToken)  ← validates the just-created session    │
│    Build full app HTML (reuse existing builder):                          │
│      ├─ Embeds _sessionToken, user email, version                         │
│      ├─ FIRST: inline postMessage gas-session-created (with appLoaded:    │
│      │   true) — fires synchronously before any async calls               │
│      ├─ THEN: google.script.run.signAppMessage() → posts gas-auth-ok     │
│      └─ Registers all listeners (version-check, user-activity, etc.)      │
│                                                                           │
│  IF failure:                                                              │
│    Return error stub (same as current — unchanged)                        │
└───────────────────────────────────────────────────────────────────────────┘
    │
    ▼  HTML receives gas-session-created (with appLoaded: true):
    │    ├─ _importHmacKey(messageKey)
    │    ├─ saveSession(token, email)
    │    ├─ _pendingSessionShow = {email, displayName}
    │    └─ SKIP iframe reload (appLoaded flag detected)
    │
    ▼  HTML receives gas-auth-ok (from same iframe — no reload):
    │    ├─ showApp(email, displayName)
    │    ├─ startCountdownTimers()
    │    ├─ startHeartbeat()
    │    └─ broadcastTabClaim()
    │
    ▼  ✅ APP VISIBLE (one load instead of two)
```

**Total: 1 `doGet()` + 1 `google.script.run` = 2 GAS executions per login**

### Why the message ordering is safe

The `gas-session-created` postMessage **must** arrive before `gas-auth-ok` because:
1. `window.top.postMessage()` is synchronous — fires immediately
2. `google.script.run.signAppMessage()` is async — requires a server round-trip
3. The inline script executes `postMessage` first, then calls `google.script.run`
4. Therefore the parent imports the HMAC key before needing to verify the `gas-auth-ok` signature

### Summary table

| Path | Before | After | Change |
|------|--------|-------|--------|
| **Standard** (URL exchange) | 2 `doGet()` + 1 `google.script.run` = 3 | 1 `doGet()` + 1 `google.script.run` = 2 | **−1 `doGet()`** |
| **HIPAA** (postMessage) | 2 `doGet()` + 2 `google.script.run` = 4 | **Unchanged** | None |
| **Session resume** (both) | 1 `doGet()` + 1 `google.script.run` = 2 | **Unchanged** | None |

## 4. Implementation Steps

### Step 1: Extract shared app HTML builder (GAS — pure refactor)

**File:** `testauth1.gs` — extract lines 1423–1604 into a reusable function

```javascript
// NEW: Shared app HTML builder
// Called by both ?exchangeToken (fresh login) and ?session= (resume/reclaim)
function buildAppHtml(sessionToken, session, appMsgKey, sessionCreatedPayload) {
  // sessionToken     — the session token string
  // session          — {email, displayName, needsReauth, ...} from validateSession()
  // appMsgKey        — messageKey for HMAC signing
  // sessionCreatedPayload — JSON string or null
  //   Non-null: fresh login — inject gas-session-created postMessage before signAppMessage
  //   Null: session resume — no gas-session-created needed

  // ... existing app HTML builder code (lines 1423–1604) ...
  // With one addition: if sessionCreatedPayload is non-null, prepend it as an
  // inline postMessage at the top of the <script> block (see Step 3)

  return HtmlService.createHtmlOutput(html)
    .setTitle(TITLE)
    .setXFrameOptionsMode(HtmlService.XFrameOptionsMode.ALLOWALL);
}
```

**Risk:** Low — pure refactor, no behavioral change.

### Step 2: URL exchange path returns full app HTML on success (GAS)

**File:** `testauth1.gs` — modify lines 1328–1358 (the `?exchangeToken` path)

**Before:** Always returns the tiny postMessage stub.

**After:**
```javascript
if (result.success) {
  // Validate the just-created session
  var session = validateSession(result.sessionToken);

  // Build gas-session-created payload with appLoaded flag
  var sessionCreatedPayload = JSON.stringify({
    type: "gas-session-created",
    success: true,
    sessionToken: result.sessionToken,
    email: result.email || "",
    displayName: result.displayName || "",
    absoluteTimeout: result.absoluteTimeout || 0,
    messageKey: result.messageKey || "",
    appLoaded: true  // ← NEW: tells parent to skip iframe reload
  });

  // Return full app HTML with gas-session-created embedded
  return buildAppHtml(result.sessionToken, session, result.messageKey, sessionCreatedPayload);
} else {
  // Error path — return stub with error (unchanged from current code)
  var errorPayload = JSON.stringify({
    type: "gas-session-created",
    success: false,
    error: result.error || "exchange_failed"
  });
  var exchangeHtml = '<!DOCTYPE html><html><body><script>'
    + 'try { window.top.postMessage(' + errorPayload + ', '
    + JSON.stringify(PARENT_ORIGIN) + '); } catch(e) {}'
    + '</' + 'script></body></html>';
  return HtmlService.createHtmlOutput(exchangeHtml)
    .setTitle(TITLE)
    .setXFrameOptionsMode(HtmlService.XFrameOptionsMode.ALLOWALL);
}
```

**Risk:** Medium — new control flow on the success path. Error path is unchanged.

### Step 3: Inject gas-session-created into app HTML when provided (GAS)

**File:** `testauth1.gs` — inside `buildAppHtml()`

When `sessionCreatedPayload` is non-null, add this at the **top** of the inline `<script>` block, before the `signAppMessage` call:

```javascript
// If this is a fresh login (not session resume), notify the parent FIRST
${sessionCreatedPayload
  ? 'try { window.top.postMessage(' + sessionCreatedPayload
    + ', ' + JSON.stringify(PARENT_ORIGIN) + '); } catch(e) {}'
  : '// Session resume — no gas-session-created needed'}
```

This fires synchronously before `google.script.run.signAppMessage()` (which is async), guaranteeing the parent imports the HMAC key before `gas-auth-ok` arrives.

**Risk:** Low — additive injection controlled by a parameter.

### Step 4: Skip iframe reload when appLoaded is true (HTML)

**File:** `testauth1.html` — modify the `gas-session-created` handler (around line 1516)

**Before (line 1516):**
```javascript
gasApp.src = baseUrl + '?session=' + encodeURIComponent(data.sessionToken);
```

**After:**
```javascript
if (data.appLoaded) {
  // Optimized single-load path — app HTML already loaded in iframe.
  // gas-auth-ok will arrive from the same iframe via google.script.run.
  _expectingSession = false;
} else {
  // HIPAA postMessage path or fallback — reload iframe with session token
  _expectingSession = true;
  gasApp.src = baseUrl + '?session=' + encodeURIComponent(data.sessionToken);
}
```

**Risk:** Low — additive guard. When `appLoaded` is absent (HIPAA path, session resume), behavior is identical to current code.

### Step 5: Update `?session=` path to use shared builder (GAS — pure refactor)

**File:** `testauth1.gs` — modify lines 1402–1607

Replace the inline app HTML builder with:
```javascript
return buildAppHtml(sessionToken, session, appMsgKey, null);
```

The `null` means no `gas-session-created` is injected — this path is for session resume, not fresh login.

**Risk:** Low — pure refactor, same output.

## 5. Security Invariants

Every security mechanism from Phases 1–9 must remain functional. This checklist verifies that:

| # | Invariant | How to verify | Broken if |
|---|-----------|---------------|-----------|
| 1 | HMAC key imported via `_importHmacKey()` | `_hmacKeySet === true`, `_hmacKey` is CryptoKey | Messages can't be verified |
| 2 | First-write-wins: duplicate `gas-session-created` rejected | Post forged message → `_hmacKey` unchanged | Key overwrite attack |
| 3 | Origin validation on all postMessages | Forged cross-origin message → rejected at Layer 0 | Cross-origin injection |
| 4 | `gas-auth-ok` signature verified via HMAC-SHA256 | Tampered message → rejected | Message forgery |
| 5 | Session timer immutability | `window.SERVER_SESSION_DURATION = 999` → ignored | Timer tampering |
| 6 | `showApp()` deferred until `gas-auth-ok` | Forged `gas-session-created` alone → app stays hidden | UI spoofing |
| 7 | `_expectingSession` guard | Stale `gas-needs-auth` during exchange → ignored | Auth wall flash |
| 8 | BroadcastChannel: no credentials in messages | BC message has only `type` + `tabId` | Token leak |
| 9 | Token-in-URL elimination (heartbeat, signout) | Network tab: no tokens in URLs for those paths | History exposure |
| 10 | Session resume still works | Refresh page → app loads from localStorage | Login loop |
| 11 | "Use Here" tab reclaim still works | Tab B claims → Tab A reclaims → visible | Multi-tab broken |
| 12 | Sign-out clears HMAC key | After sign-out: `_hmacKey === null` | Key persistence |

### Message ordering guarantee

The optimization depends on `gas-session-created` arriving before `gas-auth-ok`. This is guaranteed because:
- `postMessage()` is synchronous (fires immediately when the inline script runs)
- `google.script.run` is async (requires a GAS server round-trip of ~1–3 seconds)
- The inline script runs `postMessage` first, then calls `google.script.run`

Even in the fallback case (where `signAppMessage()` fails and the failure handler sends an unsigned `gas-auth-ok` with `messageKey`), `_hmacKeySet` is already `true` from `gas-session-created`, so the duplicate import is safely skipped.

## 6. HIPAA Compliance Analysis

### Does this change affect the HIPAA (postMessage) path?

**No.** The HIPAA path is completely unchanged — same 2 loads, same 4 GAS executions, same behavior.

### Does this change affect HIPAA compliance on the standard path?

The standard path is not the HIPAA-certified configuration (`TOKEN_EXCHANGE_METHOD: 'postMessage'` is the HIPAA preset). However, the optimization is neutral-to-positive for security on any path:

| Safeguard | Before | After |
|-----------|--------|-------|
| **Transmission security** (§164.312(e)(1)) | HTTPS (GAS enforces TLS) | Unchanged |
| **Access controls** (§164.312(a)(1)) | OAuth + session + ACL | Unchanged |
| **Audit controls** (§164.312(b)) | `auditLog()` in exchange | Unchanged |
| **Integrity** (§164.312(c)(1)) | HMAC-SHA256 signing | Unchanged |
| **Authentication** (§164.312(d)) | Google OAuth + domain + ACL | Unchanged |
| **Minimum necessary** (§164.502(b)) | 2 network round-trips | **Improved** — 1 fewer round-trip |

### Session token URL exposure

| Scenario | Before | After |
|----------|--------|-------|
| Standard login | Token appears in `?session=` URL on Load 2 | **Token never appears in URL** — only in `gas-session-created` postMessage |
| Session resume | Token in `?session=` URL | Unchanged (resume still uses `?session=`) |
| HIPAA login | Token in `?session=` URL on Load 1 | Unchanged |

The standard login path gets a net improvement — one fewer instance of token-in-URL.

## 7. GAS Quota Impact

### How GAS quotas work (verified March 2026)

Per the [official Google Apps Script Quotas page](https://developers.google.com/apps-script/guides/services/quotas):

- **`doGet()`/`doPost()` are NOT subject to trigger quotas** — they are web app entry points
- **`google.script.run` counts as a server-side execution** — subject to per-execution time limit (6 min) and daily quotas
- **Quotas are per-account** (not per-script) — all scripts under one Google account share the same pool
- **URL Fetch:** Consumer = 20,000/day; Workspace = 100,000/day
- **Execution time:** 6 min per execution (all account types)

### Savings per login

| Operation | Before (standard) | After (standard) |
|-----------|--------------------|-------------------|
| `doGet(?exchangeToken=)` | 1 | 1 |
| `doGet(?session=)` | 1 | **0 (eliminated)** |
| `google.script.run.signAppMessage()` | 1 | 1 |
| **Total** | **3** | **2** |

### CacheService impact

The optimization adds one `validateSession()` call to the `?exchangeToken` path (to validate the just-created session before building app HTML). This is a CacheService read + write — but it's offset by eliminating the entire Load 2, which did the same `validateSession()` call. **Net CacheService calls: unchanged.**

### Projected savings at scale

| Users/day | Before (executions) | After (executions) | Saved |
|-----------|---------------------|--------------------|-------|
| 10 | 30 | 20 | 10/day |
| 50 | 150 | 100 | 50/day |
| 100 | 300 | 200 | 100/day |

## 9. Rollback Plan

### Quick rollback (GAS-only)

The `appLoaded` flag in the HTML handler makes rollback trivial:

1. **Revert `testauth1.gs`** — restore the `?exchangeToken` path to return the postMessage stub (lines 1342–1357 original)
2. **Redeploy GAS**
3. **Done** — no HTML change needed

When the GAS side stops sending `appLoaded: true`, the HTML handler falls through to the `else` branch and reloads the iframe — exactly the current behavior.

### Feature flag option

Add a config toggle for gradual rollout:

```javascript
// In AUTH_CONFIG presets:
SINGLE_LOAD_AUTH: true,  // true = optimized, false = legacy 2-load

// In ?exchangeToken path:
if (result.success && AUTH_CONFIG.SINGLE_LOAD_AUTH) {
  return buildAppHtml(result.sessionToken, session, appMsgKey, sessionCreatedPayload);
} else if (result.success) {
  // Legacy: return postMessage stub
  return HtmlService.createHtmlOutput(exchangeHtml)...
}
```
