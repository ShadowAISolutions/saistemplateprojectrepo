Setup & Configuration localStorage
  1. Create a new Apps Script project at script.google.com and paste your .gs file contents
  2. In Project Settings, enable Show appsscript.json manifest file and set its contents to: 
    {
  "timeZone": "America/New_York",
  "runtimeVersion": "V8",
  "dependencies": {},
  "webapp": {
    "executeAs": "USER_DEPLOYING",
    "access": "ANYONE_ANONYMOUS"
  },
  "exceptionLogging": "STACKDRIVER",
  "oauthScopes": [
    "https://www.googleapis.com/auth/script.projects",
    "https://www.googleapis.com/auth/script.external_request",
    "https://www.googleapis.com/auth/script.deployments",
    "https://www.googleapis.com/auth/spreadsheets",
    "https://www.googleapis.com/auth/script.send_mail"
  ]
}
  3. Create or use a GCP project where you have Owner access. If you don't have one, create a new project at console.cloud.google.com
  4. Enable the Apps Script API in your GCP project
    APIs & Services → Library → search "Apps Script API" → Enable
  5. Link the GCP project in Apps Script
    Project Settings → Google Cloud Platform (GCP) Project → Change project
  6. Enable the Apps Script API at script.google.com/home/usersettings
  7. Deploy as a Web app
    Deploy → New deployment → Select type: Web app → Execute as: Me → Who has access: Anyone
    Shown in browser tab and GAS UI
  8. Copy the Deployment ID and paste it below
    From Apps Script: Deploy → Manage deployments
  9. Set GITHUB_TOKEN in Script Properties
    Project Settings → Script Properties → Add: Key = GITHUB_TOKEN, Value = your fine-grained personal access token
    To create a token: GitHub.com → Settings → Developer settings → Personal access tokens → Fine-grained tokens → Generate new token. Under "Repository access" select Public repositories (read-only). No additional permissions are needed. Copy the github_pat_... value.
    Reuse across projects: The same token works for all your GAS projects — you don't need a new one per repo. Just paste the same token into each project's Script Properties.
  10. Run any function from the Apps Script editor to trigger the initial OAuth authorization prompt
  11. (Optional) If using Google Sheets: create a spreadsheet and enter the details below
    Google Sheets ID for version tracking
    Tab name in the spreadsheet
  12. (Optional) If using an installable trigger for sheet caching:
    Apps Script editor → Triggers → + Add Trigger → Function: onEditWriteB1ToCache, Event source: From spreadsheet, Event type: On edit
  13. (Optional) Add a notification sound from Google Drive
    Google Drive file ID for notification sound
Values saved in browser localStorage for instant testing — use Copy Config to export for your repo
